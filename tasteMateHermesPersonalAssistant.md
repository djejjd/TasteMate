# TasteMate Hermes 自用型方案论证

## 一、当前阶段目标

当前阶段不是设计一个完整推荐系统，也不是扩展成面向大众的产品。当前要论证的是：

```text
TasteMate 是否值得作为 Hermes 的个人偏好层来做。
```

更具体地说，要判断三件事：

```text
方向是否合理
技术上是否可行
做出来能带来什么实际收益
```

因此，这一版不追求完整产品形态，而是把范围控制在“自用型 Hermes 个人推荐助手”的最小可验证方案。

---

## 二、核心定位

TasteMate 不应该先做成一个独立推荐平台，而应该先做成 Hermes 的个人品味记忆与推荐重排层。

一句话定义：

```text
TasteMate 是 Hermes 的个人品味记忆与推荐重排层，
用于让 Hermes 在信息获取、内容筛选和推荐任务中更符合我的长期偏好。
```

Hermes 负责执行任务：

```text
搜索信息
收集候选内容
整理结果
生成回答
```

TasteMate 负责偏好判断：

```text
我可能更喜欢什么
什么内容不适合我
哪些结果应该排前面
为什么推荐或不推荐
我的反馈应该如何更新偏好
```

这个定位的好处是：TasteMate 不需要一开始承担全网搜索、内容平台、多用户协同过滤或商业排序，只要先验证它能不能让 Hermes 的输出更贴近我的口味。

---

## 三、为什么这个方向合理

Hermes 本身可以完成搜索、总结和任务执行，但它默认不知道我的长期偏好。即使它能根据当前对话推断一些偏好，也很难稳定记住：

```text
我长期关注哪些主题
我喜欢什么信息密度
我偏好什么表达风格
我不喜欢哪些内容来源或内容类型
我最近正在关注什么项目
```

TasteMate 的价值在于把这些偏好沉淀成一个可复用的上下文层。以后 Hermes 每次找新闻、文章、项目、电影、资料时，都可以先调用这个偏好层，再给出更贴近我的结果。

这比单独做一个“推荐系统”更适合当前阶段，因为自用场景下最有价值的不是推荐数量，而是：

```text
减少无效信息
提高筛选效率
让推荐结果更像我自己会选的东西
让 Hermes 逐渐形成稳定的个人偏好记忆
```

---

## 四、Hermes 是先按偏好找，还是先找完再筛选

更合理的策略不是二选一，而是分层使用偏好：

```text
搜索阶段：偏好轻度参与
筛选阶段：偏好强度参与
反馈阶段：偏好持续学习
```

也就是说，Hermes 不应该一开始就完全按我的偏好去找。这样容易让搜索范围过窄，漏掉新内容，也容易形成信息茧房。

更合理的流程是：

```text
用户指令
        ↓
Hermes 理解任务
        ↓
Hermes 先根据任务本身收集一批候选
        ↓
TasteMate 提供偏好关键词、负向偏好、当前关注方向
        ↓
Hermes 可以用这些偏好补充搜索
        ↓
合并候选结果
        ↓
TasteMate 统一打分、排序、解释
        ↓
用户选择或反馈
        ↓
反馈写回 Taste Profile
```

这个流程兼顾了两件事：

```text
候选召回保持足够广度
最终排序体现个人偏好
```

所以 TasteMate 第一阶段的重点不是替代 Hermes 搜索，而是给 Hermes 的候选结果做个人化筛选和重排。

---

## 五、评分为什么重要

评分是 TasteMate 的核心，因为它决定两件事：

```text
Hermes 找到的一批候选里，哪些排前面
我的选择和反馈如何转化为长期偏好
```

第一阶段不需要复杂机器学习模型，应该先做一个可解释的加权评分模型。

建议评分由五部分组成：

```text
内容质量
个人口味匹配
当前任务匹配
新鲜度
历史反馈
```

可以先用这样的思路：

```text
final_score =
  quality_score       * 0.25
+ taste_match_score   * 0.35
+ context_match_score * 0.20
+ novelty_score       * 0.10
+ feedback_score      * 0.10
```

这不是最终算法，只是第一版论证模型。它的价值在于可解释、可调试、可观察。

每个候选内容的评分结果应该能说明：

```text
为什么推荐
为什么扣分
它匹配了哪些个人偏好
它不匹配当前任务的地方是什么
它是否只是质量高但不适合我
```

这比只输出一个“推荐/不推荐”更有价值，也方便后续根据实际使用调整权重。

---

## 六、反馈是否能加强个人偏好学习

可以，而且这应该是 TasteMate 的核心闭环。

每次 Hermes 给出结果后，我的选择都可以作为反馈信号：

| 行为 | 含义 |
| --- | --- |
| 收藏 | 强正反馈 |
| 继续追问 | 强兴趣 |
| 选择/点击 | 中等正反馈 |
| 停留/阅读 | 弱正反馈 |
| 跳过 | 弱负反馈 |
| 不相关 | 当前任务匹配差 |
| 不喜欢 | 个人口味不匹配 |
| 以后少推荐 | 强负反馈 |

但这里有一个关键原则：

```text
不要把一次选择立刻当成长期偏好。
```

更稳妥的学习方式是分层更新：

```text
一次选择
→ 记录为 interaction

多次类似选择
→ 提升为 temporary interest

长期反复出现
→ 写入 stable preference
```

例如，我最近频繁选择“AI 推荐系统”相关内容，这可能只是因为当前正在做 TasteMate，不一定代表长期偏好。因此第一阶段应该先记入：

```text
current_focus
```

只有当类似行为长期重复出现，才升级到：

```text
stable_preferences
```

这样可以避免 TasteMate 学歪，也能区分“我长期喜欢什么”和“我最近正在做什么”。

---

## 七、第一版 Taste Profile 应该保存什么

第一版画像不需要太复杂，但必须能区分不同层级的偏好。

建议先保存：

```json
{
  "stable_preferences": {
    "topics": [],
    "styles": [],
    "media": []
  },
  "negative_preferences": {
    "topics": [],
    "styles": [],
    "sources": []
  },
  "current_focus": {
    "projects": [],
    "topics": []
  },
  "interaction_history": []
}
```

这里的重点不是字段多，而是边界清楚：

```text
stable_preferences：长期偏好
negative_preferences：明确不喜欢或应减少推荐的内容
current_focus：当前项目和短期兴趣
interaction_history：每次选择、跳过、反馈的原始记录
```

有了这几类数据，TasteMate 就能开始形成一个基础的个人偏好记忆。

## 八、可行性判断

这个方向在技术上是可行的，因为第一版不依赖大规模训练，也不依赖多用户数据。

第一版可以只需要：

```text
一个本地 Taste Profile 文件
一份交互反馈日志
一个候选内容评分函数
一个 LLM 解释层
一个 Hermes 可调用的接口
```

候选内容可以先由 Hermes 提供。TasteMate 只做：

```text
结构化候选内容
读取个人画像
计算五维评分
生成排序结果
解释推荐理由
记录反馈
```

这说明项目的第一阶段可以很轻，不需要先做：

```text
协同过滤
多用户系统
大型向量数据库
复杂图推荐
完整前端
账号系统
```

技术复杂度主要在三处：

| 难点 | 说明 | 第一阶段处理方式 |
| --- | --- | --- |
| 偏好抽取 | LLM 可能总结过度 | 所有偏好带来源和置信度 |
| 评分权重 | 初始权重不一定准确 | 手动可调，先观察结果 |
| 反馈学习 | 容易把短期行为当长期偏好 | 分 interaction/current_focus/stable_preferences 三层 |

这些难点都可以通过小范围自用数据逐步校准，不需要一开始做重架构。

## 九、预期收获

如果这个方向成立，TasteMate 能带来的收益主要有四类。

## 1. 信息筛选效率提升

Hermes 找到 30 个候选时，TasteMate 可以帮我优先看最符合偏好的 5 个，减少无效阅读和反复筛选。

## 2. 推荐结果更稳定

不是每次都靠当前 prompt 临时描述偏好，而是把长期偏好沉淀下来，持续影响 Hermes 的输出。

## 3. 个人偏好资产沉淀

我的选择、收藏、跳过、不喜欢都会变成可积累的数据。时间越久，TasteMate 越能知道：

```text
我真正喜欢什么
我只是阶段性关注什么
我明确不想再看到什么
```

## 4. Hermes 更像个人助手

Hermes 不只是执行命令，而是逐渐带有个人判断：

```text
这个虽然热门，但不太适合你
这个信息密度更高，可能更符合你的风格
这个和你最近做的 TasteMate/Hermes 方向相关
这个主题你之前多次跳过，可以降低优先级
```

这就是 TasteMate 对 Hermes 的实际增强。

---

## 十、当前阶段的结论

这个方向值得继续论证和小范围验证。

最合理的下一步不是扩功能，而是做一个最小验证闭环：

```text
Hermes 收集候选
        ↓
TasteMate 打分重排
        ↓
输出推荐理由
        ↓
我选择或反馈
        ↓
TasteMate 更新个人偏好
```

如果这个闭环能在几个真实任务里明显提升结果质量，就说明方向成立。

当前阶段的判断：

```text
方向：合理
可行性：高
第一阶段范围：应保持很小
最大价值：让 Hermes 从通用助手变成更懂我的个人助手
最关键模块：评分与反馈学习
```
